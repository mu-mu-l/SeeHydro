# SeeHydro 推理与结果查看手册

这份文档解决的问题是：你训练完模型以后，怎么继续往下走。

## 1. 推理是什么

推理就是：

- 把训练好的模型权重
- 用到新的切片图像上
- 让模型输出分割掩膜或检测结果

## 2. 仓库里的推理入口

核心代码在：

- [`inference.py`](/tmp/SeeHydro/src/seehydro/models/inference.py)

这个文件里已经有统一推理管线：

- `run_segmentation()`
- `run_detection()`
- `run_full_pipeline()`

## 3. 分割推理会输出什么

按当前代码逻辑：

- 输入：切片目录里的 `.tif`
- 输出：每张切片对应一张单通道掩膜

输出目录会保存预测结果掩膜文件。

## 4. 检测推理会输出什么

按当前代码逻辑：

- 输入：切片目录里的 `.tif`
- 输出：每张切片的检测框信息

结果是字典形式，里面会有每个切片对应的目标列表。

## 5. 你训练完以后怎么理解下一步

训练完不代表项目结束。

后面通常还要：

1. 先看推理图有没有识别出水面或目标
2. 再决定模型是不是值得继续加数据训练
3. 再进入参数提取

## 6. 参数提取是什么

这个项目最终目标不是只看图，而是提取工程参数。

比如：

- 渠道宽度
- 桥梁参数
- 倒虹吸/渡槽等结构物参数

也就是说，完整路线应该是：

```text
下载 -> 切片 -> 标注 -> 训练 -> 推理 -> 提参数
```

## 7. 第一阶段你怎么判断自己有没有成功

不要一开始就追求最后报表。

第一阶段成功的标准应该是：

1. 训练能跑完
2. 推理能输出掩膜或目标
3. 结果肉眼看起来不是完全乱的

只要做到这三点，就说明链路已经打通。

## 8. 你现在最合理的目标

如果你是第一次跑这个仓库，推理阶段最合理目标是：

- 先看水面能不能被分出来
- 先不追求桥梁和复杂结构物都做好
- 先用小范围数据验证方法可行

## 9. 一句话结论

推理不是结束，而是训练后的验证步骤。

你最先要看的不是“最终报告漂亮不漂亮”，而是：

- 模型有没有学到东西
- 分割结果是不是大致合理
- 后面还有没有必要继续扩数据集
